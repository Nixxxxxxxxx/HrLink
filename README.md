# HrLink

Прототип HR-платформы с фокусом на опросы сотрудников: от создания опроса до прохождения и аналитики.

Текущая кодовая база стартует с импортированной Figma Make версии news-прототипа и используется как временный рабочий shell, от которого мы дальше перейдем к HR survey flow.

## Цель проекта

Собрать быстрый, визуально точный и логически цельный прототип, который можно:

- показывать как рабочую демо-версию;
- использовать для проверки продуктовой логики;
- постепенно углублять до production-grade архитектуры.

## Основные модули

- Создание опросов
- Прохождение опросов сотрудниками
- Аналитика по конкретному опросу
- Аналитика по компании в целом

## Принципы проекта

- Верстка макетов должна быть максимально близкой к 1:1.
- Любой экран проектируется не как картинка, а как часть реального пользовательского сценария.
- Логика важна не меньше визуала: состояния, ошибки, пустые экраны, ограничения и роли продумываются сразу.
- Код должен быть достаточно чистым и структурированным, чтобы проект можно было масштабировать без переписывания с нуля.
- В `main` держим рабочую демо-версию.
- Не используем реальные персональные данные сотрудников.

## Текущая база приложения

Сейчас в корне репозитория лежит импортированный `React + Vite + Tailwind` проект из Figma Make.

В этой версии уже есть рабочие паттерны для:

- создания новости;
- просмотра новости;
- редакторских экранов;
- ленты, боковых панелей и связанных UI-состояний.

На текущем этапе этот код не переименовывается и не очищается от news-логики. Сначала держим его как рабочую стартовую основу, затем постепенно переводим в HR-домен.

## Локальный запуск

После установки Node.js:

```bash
npm install
npm run dev
```

Production build:

```bash
npm run build
```

## Ближайшая цель

Собрать основу продукта вокруг survey flow:

1. Создание опроса HR/администратором.
2. Настройка аудитории, анонимности, сроков и логики вопросов.
3. Прохождение опроса сотрудником.
4. Просмотр результатов и аналитики.
