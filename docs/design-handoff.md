# Правила работы с макетами

## Как передавать макеты

Для каждого нового экрана или блока лучше передавать:

- ссылку на Figma;
- конкретный node или frame;
- сценарий, к которому относится экран;
- приоритет: pixel-perfect или допустимы небольшие адаптации;
- состояния, если они есть отдельно в макете.

## Что важно передавать вместе с макетом

- роль пользователя;
- цель экрана;
- основной сценарий;
- крайние состояния;
- есть ли мобильная версия;
- есть ли интерактивные состояния: hover, active, disabled, error.

## Как я буду работать с макетом

- сначала читаю экран как продуктовый сценарий;
- потом перевожу его в структуру интерфейса;
- затем собираю точную верстку;
- после этого достраиваю состояния и логику, которых не видно в статичном макете.

## Если макет неполный

Если в макете нет части логики, по умолчанию я:

- сохраняю визуальный язык макета;
- достраиваю недостающие состояния;
- фиксирую допущения;
- задаю вопрос только там, где решение влияет на архитектуру, данные или основной сценарий.

## Цель handoff

Макет должен быть не просто референсом для внешнего вида, а точкой входа в работающий экран с логикой, состояниями и качественной структурой кода.
