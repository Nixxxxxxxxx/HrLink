# Product Scope

## Ядро продукта

Продукт состоит из двух больших контуров:

- контур администрирования и создания опросов;
- контур участия и анализа результатов.

## Контур 1. Создание опросов

Первый обязательный сценарий:

1. HR открывает список опросов.
2. Создает новый опрос.
3. Настраивает название, описание, цель и сроки.
4. Добавляет вопросы.
5. Настраивает типы вопросов.
6. Выбирает аудиторию.
7. Настраивает анонимность и правила прохождения.
8. Сохраняет черновик или публикует.

Что должно поддерживаться:

- черновик;
- публикация;
- scheduled publish;
- дедлайн;
- обязательность прохождения;
- анонимный или неанонимный режим;
- предпросмотр.

## Контур 2. Прохождение опросов

Второй обязательный сценарий:

1. Employee видит список доступных опросов.
2. Открывает опрос.
3. Понимает контекст: срок, анонимность, примерное время прохождения.
4. Отвечает на вопросы.
5. Сохраняет прогресс, если это допустимо.
6. Отправляет ответы.
7. Видит подтверждение завершения.

Что должно поддерживаться:

- прогресс;
- валидация;
- блокировка повторной отправки;
- состояние "уже пройдено";
- состояние "дедлайн прошел";
- пустое состояние, если активных опросов нет.

## Контур 3. Аналитика по опросу

Экран должен отвечать на вопросы:

- сколько людей приглашено;
- сколько начали;
- сколько завершили;
- каков completion rate;
- какие ответы доминируют;
- какие сегменты отличаются;
- где проблемные зоны.

## Контур 4. Аналитика по компании

Экран должен отвечать на вопросы:

- как меняется engagement;
- какие департаменты проседают;
- где низкая вовлеченность;
- как распределяются результаты по командам;
- какие survey themes повторяются.

## Минимальный набор question types

- single choice
- multiple choice
- rating scale
- NPS-like scale
- open text
- yes/no

## Что пока не делаем в первой версии

- сложную автоматизацию кампаний;
- полноценную оргструктуру enterprise-уровня;
- тонкие права доступа по множеству политик;
- интеграции со сторонними HRIS;
- продвинутый конструктор условной логики вопросов.

## Definition of Done для модуля

Модуль считается готовым, если:

- у него есть понятный user flow;
- ключевые состояния покрыты;
- визуал согласован с макетом;
- данные выглядят реалистично;
- сценарий можно показать в демо без пояснений на словах.
